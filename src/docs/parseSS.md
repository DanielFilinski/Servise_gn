# Инструкция: Парсинг HTML-уроков СШ → SS+.json

## Обзор

Каждый HTML-файл (`L01.html`, `L02.html`, ...) содержит **один урок недели** (воскресенье–пятница).
Результат — массив JSON-объектов, **один объект на каждый день** урока.

**Пути:**
- Входные файлы: `fileForParse/SS/<YYYY-MM>/L01.html`, `L02.html`, ...
- Результат: `ResultParse/SS/SS+.json`

---

## Структура выходного JSON

### Объект дня (верхний уровень)

```json
{
  "id": 1,
  "date": "2025-12-27 00:00:00.000",
  "name": "Страдающий, но не покинутый Богом",
  "bookNumber": 1,
  "lessonNumber": 1,
  "isFirstLesson": true,
  "content": [...]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `id` | int | Сквозной ID, начиная с 1 |
| `date` | string | Дата дня: `"YYYY-MM-DD 00:00:00.000"` |
| `name` | string | Заголовок дня (из `СШ_DAY_Lesson-Day-header`) |
| `bookNumber` | int | Номер недели урока |
| `lessonNumber` | int | Всегда `1` |
| `isFirstLesson` | bool | Только у первого объекта (id=1) |
| `content` | array | Массив контентных блоков |

---

## Типы контентных блоков (content[])

```json
{"id": 1, "type": "bibleVersesTitle",  "text": "Библейские стихи для исследования:"}
{"id": 2, "type": "mainVerses",        "text": "...", "links": [...]}
{"id": 3, "type": "AdditionalVerses",  "text": "...", "links": [...]}
{"id": 4, "type": "memoryVerseTitle",  "text": "Памятный текст:"}
{"id": 5, "type": "memoryVerse",       "text": "«...» (Флп. 4:4).", "links": [...]}
{"id": 6, "type": "mainText",          "text": "...", "links": [...]}
{"id": 7, "type": "question",          "text": "...", "links": [...]}
```

> Поле `links` присутствует только если в тексте есть библейские ссылки.

## Маппинг HTML CSS-классов → JSON

### Заголовок урока (один раз на файл)

| CSS-класс | Span/элемент | Назначение |
|---|---|---|
| `СШ_Lesson-Number` → `span.Header_Lesson` | "Урок 1" | Номер урока |
| `СШ_Lesson-Number-DATE` | "27 декабря — 2 января" | Вычислить `date` первого дня |
| `СШ_Lesson-Name` | "Страдающий, но..." | Название недели (для `bookNumber`) |

### Вводная часть (первый день)

| CSS-класс | JSON type |
|---|---|
| `СШ_Lesson-intro-БТИ` | `bibleVersesTitle` |
| `СШ_Lesson-intro-БТИ-текст` (содержит "Основные отрывки") | `mainVerses` |
| `СШ_Lesson-intro-БТИ-текст` (содержит "Дополнительные тексты") | `AdditionalVerses` |
| `СШ_Lesson-intro-БТИ-текст` (иначе) | `mainText` |
| `СШ_Lesson-intro-ПС` | `memoryVerseTitle` |
| `СШ_Lesson-intro-ПС-текст` | `memoryVerse` |

### Ежедневные разделы

| CSS-класс | Описание | JSON |
|---|---|---|
| `СШ_DAY_Lesson-Day` | Граница нового дня: span`Lesson-header_Day_bold` + span`Lesson-header_Lesson-Day-number` + текст месяца | Новый объект дня, вычислить дату |
| `СШ_DAY_Lesson-Day-header` → span.bold | Заголовок дня | `name` объекта |
| `основной-абзац` | Основной абзац | `mainText` |
| `СШ_Question__1_Question_continued` | Вопрос | `question` |
| `СШ_Question__3_Question_w-o_spacing` | Вопрос (альт.) | `question` |
| `СШ_Question__lines-w-o-space` | Пустые строки для ответа | `question` (пустой текст) |
| `list_questions_for_discus-SIMPLE` | Пункт обсуждения | `mainText` (без номера) |

## Правила трансформации

### 1. Очистка текста

- `&#160;` → пробел ` `
- `&#9;` → удалить
- `<span class="hiddenNumber ...">` → удалить элемент полностью
- Все `<span>` → извлечь только текстовое содержимое
- Множественные пробелы → один пробел
- Кавычки `«»` и тире `—` сохранять как есть

### 2. Парсинг дат

**Из `СШ_Lesson-Number-DATE`:** формат `"DD месяц — DD месяц"`

1. Год берётся из пути файла: `fileForParse/SS/YYYY-MM/`
2. При переходе через год (декабрь→январь): декабрьская дата → год из пути минус 1
3. Пример: путь `2026-01`, строка `"27 декабря — 2 января"` → `2025-12-27 00:00:00.000`

**Из `СШ_DAY_Lesson-Day`:** определить дату каждого дня по числу и месяцу.

**Маппинг месяцев:**

### 3. Парсинг библейских ссылок

**Паттерны поиска в тексте:**
- В скобках: `(Книга. Гл:Стих[–Стих][; ...])`
- Инлайн: `Книга. Гл:Стих`

**Структура объекта ссылки:**
```json
{
  "text": "Флп. 1:1–3",
  "data": [
    {
      "bookNumber": 570,
      "chapter": [1],
      "verses": [1, 2, 3]
    }
  ]
}
```

**Правила:**
- Диапазон `1–3` → развернуть в `[1, 2, 3]`
- Несколько глав `4:7–12; 6:3–7` → два объекта в `data[]`
- Одна ссылка с несколькими книгами → один объект `links[]` или отдельные — по контексту

## Порядок обработки одного HTML-файла

1. **Определить год** из пути: `fileForParse/SS/YYYY-MM/`
2. **Прочитать дату** из `СШ_Lesson-Number-DATE` → вычислить дату начала урока
3. **Прочитать заголовок** из `СШ_Lesson-Name` → установить `bookNumber`
4. **Разобрать введение** (первый день): bibleVersesTitle, mainVerses/mainText, memoryVerseTitle, memoryVerse → добавить в `content[]` первого объекта
5. **Для каждого `СШ_DAY_Lesson-Day`**:
   - Создать новый объект дня
   - Вычислить дату из числа и месяца
   - Следующий `СШ_DAY_Lesson-Day-header` → `name`
   - Все параграфы до следующего `СШ_DAY_Lesson-Day` → `content[]`
6. **Пронумеровать id** сквозно по всем объектам (и по всем файлам в партии)

## Различия форматов 2025 vs 2026

| Аспект | 2025-02 | 2026-01 |
|---|---|---|
| Вводные секции | Разделены: `mainVerses` + `AdditionalVerses` | Объединены в `mainText` |
| Подзаголовки принципов | `СШ_DAY_Lesson-Day-SUBheader-for-begin-days` | Могут отсутствовать |
| Сложность | Больше типов | Упрощённая структура |
